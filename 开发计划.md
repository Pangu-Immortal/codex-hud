# 开发计划

## 需求概述
打造一个面向 OpenAI Codex 的 macOS 菜单栏 HUD，实时显示会话、后台 Agent、线程活动、日志告警与诊断导出，并以开源仓库形式发布。

## 复杂度评估
级别：🟡 Standard  
依赖模式：Fan-out（采集层并行） → Fan-in（HUD 聚合）

## 模块拆解

|----|------|------|---------|------|------|

## 验收标准
1. 应用可通过 `swift build` 成功编译。
2. 菜单栏 HUD 能显示运行中的 Codex 会话数、热点线程数、后台 Agent 数、最近告警。
3. 支持从 `.codex` 真实目录读取数据，而不是硬编码假数据。
4. 支持生成用于仓库展示的 PNG 截图。
5. 仓库具备 README、SEO/GEO 元信息、CI 与开源许可证。

## 风险与对策
1. Codex 本地数据库结构未来可能变更：通过容错查询与诊断导出降低风险。
2. 菜单栏场景不适合过大界面：使用滚动面板与分区卡片控制高度。
3. 后台状态难以精确定义：同时融合进程、线程更新时间、spawn 边与 agent job 状态三路信号。
